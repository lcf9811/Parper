# WAgent 与 OpenClaw 对比分析

## 1. 概述

WAgent 是一套完整的 **ReAct 模式 AI Agent 运行平台**，以 OpenClaw 为灵感起点，但在架构设计、功能完整性、生产可用性等方面进行了全面升级。本文档系统性地对比两者的差异，说明 WAgent 的领先性和优势。

---

## 2. 核心架构对比

| 维度 | OpenClaw | WAgent |
|------|----------|--------|
| **Agent 模式** | 基于规则的指令执行 | **LangGraph ReAct 循环**（Agent → Tools → Agent） |
| **运行时** | 线性执行流程 | **有状态图（StateGraph）**，支持循环、分支、条件路由 |
| **工具循环** | 不支持 | **自动循环调用**，最多 10 次（可配置），防止无限循环 |
| **知识检索** | 无内置能力 | **自动检索注入**，在 Agent 决策前自动检索知识库并注入上下文 |
| **流式输出** | 无 | **SSE 实时流式输出**，带事件缓冲和断线重放机制 |

### 2.1 ReAct 模式 vs 规则执行

OpenClaw 的执行流程本质上是**线性**的：接收输入 → 匹配规则 → 执行命令 → 返回结果。

WAgent 基于 LangGraph 实现了 **ReAct（Reasoning + Acting）循环**：

```
[START] → [knowledge_retrieval] → [agent] → [tools] → [agent] → ... → [END]
              ↑ (可选)                     ↘ no tools ↗
```

这种架构的优势在于：
- **自主决策**：Agent 自行判断是否需要调用工具、调用哪个工具、调用几次
- **多步推理**：可以连续调用多个工具，每步的结果都会反馈给 Agent 进行下一步决策
- **知识增强**：在 Agent 决策前自动检索知识库，将相关上下文注入 system prompt
- **灵活路由**：根据消息内容动态选择执行路径

## 3. 技能系统对比

| 维度 | OpenClaw | WAgent |
|------|----------|--------|
| **技能定义** | 固定脚本 | **Markdown 文件（SKILL.md）**，支持 frontmatter 元数据 |
| **多技能组合** | 不支持 | **多技能同时激活**，自动合并为多角色 system prompt |
| **技能引用文档** | 手动管理 | **自动摄取** references/、assets/ 下的 .md 文件到知识库 |
| **技能发现** | 硬编码路径 | **自动扫描** skills/ 目录，启动时同步到数据库 |
| **技能目录查询** | 无 | **skill_catalog 工具**，Agent 可自主查询可用技能 |
| **旧格式兼容** | N/A | **支持 local/*.md** 旧格式兼容迁移 |

### 3.1 多技能组合示例

当用户同时启用「联动换线评估」和「进水毒性检测」两个技能时，WAgent 会自动构建如下 system prompt：

```markdown
你同时具备以下多种能力，请根据用户的问题灵活运用：

## 角色 1：联动换线评估与执行
[SKILL.md 内容...]

## 角色 2：进水毒性检测与分级
[SKILL.md 内容...]
```

OpenClaw 每次只能执行一个预定义的任务流程，无法灵活组合多种能力。

## 4. 知识库集成对比

| 维度 | OpenClaw | WAgent |
|------|----------|--------|
| **文档入库** | 无 | **自动分块存储**（默认 500 字符/块，50 字符重叠） |
| **搜索能力** | 无 | **MySQL FULLTEXT 全文搜索**，支持相关性排序，降级到 LIKE |
| **自动检索注入** | 无 | **autoKnowledgeRetrieval 模式**，Agent 决策前自动检索 |
| **手动知识查询** | 无 | **knowledge_lookup 工具**，用户/Agent 可按关键词搜索 |
| **分块管理** | 无 | **UI 查看分块结果**，支持分块参数配置 |
| **技能文档自动入库** | 无 | **技能 references/ 自动摄取**，技能定义即知识库 |

### 4.1 搜索策略

```sql
-- 优先：FULLTEXT 全文搜索（带相关性排序）
SELECT c.*, d.title as doc_title,
       MATCH(c.content) AGAINST(? IN NATURAL LANGUAGE MODE) AS relevance
FROM knowledge_chunks c
JOIN knowledge_documents d ON c.document_id = d.id
WHERE MATCH(c.content) AGAINST(? IN NATURAL LANGUAGE MODE)
ORDER BY relevance DESC
LIMIT ?

-- 降级：LIKE 模糊匹配（FULLTEXT 不可用时自动切换）
```

## 5. Webhook 与外部集成对比

| 维度 | OpenClaw | WAgent |
|------|----------|--------|
| **外部触发** | 无 | **Webhook 接收器**，支持 POST 请求触发 Agent 执行 |
| **认证机制** | 无 | **Bearer Key + URL Token 双重认证**，每个端点独立密钥 |
| **流式回传** | 无 | **SSE 实时推送**，LLM 输出逐字回传到前端 |
| **事件缓冲** | 无 | **无客户端连接时缓冲事件**（最多 200 个），连接后自动重放 |
| **端点管理** | 无 | **UI 管理 Webhook 端点**，支持启用/禁用、选择工具/技能 |
| **立即确认** | N/A | **不等待执行完成即返回** executionId，适合异步场景 |

### 5.1 事件缓冲机制

这是 WAgent 独有的设计，解决了 webhook 场景下「执行已经开始但前端还没连接」的问题：

```
Webhook 触发 → runAgentStreaming → sseService.emit() → [无客户端连接] → 事件缓冲
                                                                    ↓
客户端连接 → sseService.registerClient() → 检测缓冲事件 → setImmediate() 重放
```

## 6. 生产可用性对比

| 维度 | OpenClaw | WAgent |
|------|----------|--------|
| **命令安全** | 无限制 | **白名单 + 黑名单双重校验**，正则匹配危险模式 |
| **工具循环限制** | 无 | **默认最多 10 次**，防止 Agent 陷入无限循环 |
| **System Prompt 长度** | 无限制 | **超过 12,000 字符自动降级**为 general_assistant，防止 token 溢出 |
| **错误恢复** | 无 | **执行状态追踪**（pending → running → completed/failed），错误可追溯 |
| **超时控制** | 无 | **命令执行 30 秒超时**，SSE 30 秒超时回退 |
| **竞态防护** | 无 | **发送锁 + 跨会话消息污染防护** |
| **历史消息限制** | 无 | **可配置最大历史消息数**，防止 context window 溢出 |
| **用户认证** | 无 | **JWT 认证 + bcrypt 密码加密**，支持角色权限 |
| **执行追踪** | 无 | **完整的 execution + execution_steps 表**，记录每步输入/输出/耗时 |

## 9. 总结：WAgent 的核心优势

### 9.1 从「脚本执行器」到「Agent 平台」

OpenClaw 本质上是一个**任务自动化脚本**，而 WAgent 是一个**完整的 Agent 运行平台**。两者的差异不是功能多少的问题，而是架构范式的根本不同：

- OpenClaw：`if-then` 规则 → 执行命令 → 返回结果
- WAgent：`ReAct` 循环 → 自主决策 → 多步推理 → 工具调用 → 知识增强 → 流式输出

### 9.2 六大核心优势

1. **智能决策能力**：基于 LangGraph 的 ReAct 模式，让 Agent 自主决定何时调用工具、调用几次、如何组合，而非硬编码的执行流程。

2. **知识驱动**：内置知识库管理系统，支持自动分块、全文搜索、自动检索注入，让 Agent 的回答建立在领域知识基础上。

3. **外部系统集成**：Webhook + SSE 流式输出 + 事件缓冲，使 WAgent 可以与 SCADA、监控系统等外部系统无缝对接。

4. **生产级可靠性**：错误恢复、超时处理、竞态防护、命令安全校验、token 限制等生产级特性，确保系统在真实环境中稳定运行。

5. **配置驱动**：模型、工具、技能、知识库均可通过 UI 配置，修改即生效，无需改动代码或重新部署。

6. **轻量部署**：仅需 MySQL + Node.js，无需 Redis、Kafka、Kubernetes 等复杂基础设施，适合边缘计算和资源受限场景。

|------|----------|--------|
| 简单的定时任务 | ✅ | ✅ |
| 需要知识检索的运维问答 | ❌ | ✅ |
| 多步骤故障诊断 | ❌ | ✅ |
| 外部系统联动（SCADA/监控） | ❌ | ✅ |
| 多领域能力组合 | ❌ | ✅ |
| 流式输出展示 | ❌ | ✅ |
| 执行过程追溯和审计 | ❌ | ✅ |
| 多用户权限管理 | ❌ | ✅ |
