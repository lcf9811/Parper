# 水处理三工序系统架构说明

## 系统概述

本系统基于规则的三工序水处理控制系统，包含进水、吸附、出水三个核心工艺段。系统采用"二备一"冗余设计原则，确保系统的高可用性和可靠性。

## 工艺流程与管道对应关系

### 1. 管路连接关系
- **进水管道1** → **转盘1** → **出水管道1**
- **进水管道2** → **转盘2** → **出水管道2**
- **进水管道3** → **转盘3** → **出水管道3**

每套管路独立运行，水质处理路径一一对应，确保处理过程的追溯性。

### 2. "二备一"运行原则
- 同一时刻只有 **2个管道** 在运行状态
- 第3个管道处于 **停止/备用** 状态
- 运行管道故障时，备用管道可自动/手动切换
- 定期轮换运行管道，均衡设备磨损

### 3. 一条线路与三工序联动切换
- **一条线路**：同一编号的 **进水管道 N → 吸附转盘 N → 出水管道 N** 视为一条物理线路，三处状态需保持一致（运行或备用）。
- **切换含义（毒性场景）**：当某条**正在运行**的线路因毒性等原因需退出时，只需指定**要关闭的线路编号**；系统**自动启用当前唯一备用线路**，无需操作员再选「启用哪一条」。
- **联动要求**：线路切换**不是**仅进水侧单独动作，而应对 **进水、吸附、出水** 三工序**同步**下发（一次操作完成三处同一编号的启停与流量/频率等状态对齐）。
- **模拟层状态**：后端在 `inlet_perception.tags.line_switch_state` 中维护当前 **运行线路列表** `running`（长度为 2）与 **备用线路** `standby`（长度为 1）；联动切换成功后，三域 `*_perception.tags` 均写入 `last_coordinated_switch` 摘要，便于追溯。

## 各工序详细说明

### 进水工序
- **入口数量**: 1个总入口
- **管道数量**: 3个进水管道
- **监测参数**（每个管道独立检测）:
  - 毒性检测 (Toxicity Detection)
  - 温度 (Temperature, ℃)
  - 进水流量 (Inlet Flow, m³/h)
- **控制逻辑**:
  - 根据进水毒性调整流量分配
  - 毒性抑制率≥60%时停止进水
  - 基于毒性水平的流量推荐值:
    - ≥60%: 停止进水
    - 50-60%: 10-30%流量
    - 30-50%: 30-50%流量
    - 20-30%: 50-70%流量
    - 10-20%: 70-90%流量
    - <10%: 90-100%流量

### 吸附工序
- **设备配置**: 3个吸附转盘
- **监测参数**（每个转盘独立监测）:
  - 温度 (Temperature, ℃)
  - 转盘频率 (Disk Frequency, Hz)
  - 输入流量 (Input Flow, m³/h)
  - 吸附饱和率 (Adsorption Saturation Rate, %)
- **运行阶段**:
  - 材料饱和率分10个阶段 (10%-85%)
  - 各阶段对应不同的毒性去除率、流量、频率、温度范围
  - 饱和率≥80%时启动再生程序
- **控制逻辑**:
  - 根据饱和率动态调整运行参数
  - 转盘频率与毒性去除率正相关
  - 温度维持在14-16℃最佳范围

### 出水工序
- **管道数量**: 3个出水管道
- **监测参数**（每个管道独立检测）:
  - 毒性检测 (Toxicity Detection, %)
  - 出水流量 (Outlet Flow, m³/h)
- **水质标准**:
  - **达标**: 毒性<10%
  - **再监测**: 毒性10-20%
  - **不达标**: 毒性>20%
- **控制逻辑**:
  - 出水毒性超标时停止对应管路
  - 启动备用管路接替处理
  - 监测与控制参数以毒性检测、出水流量为主（界面不提供「深度处理强度」单项）

## 运行逻辑

### 1. 启动流程
1. 系统初始化，加载配置
2. 确定运行管道（默认管道1、2运行，管道3备用）
3. 启动对应转盘和出水管道
4. 开始水质处理流程

### 2. 正常运行
- 运行管道处理进水，对应转盘进行吸附，最终出水
- 备用管道处于热备状态，可快速切换
- 实时监测各参数，确保水质达标

### 3. 故障处理
- **进水毒性超标**: 根据毒性抑制率调整或停止进水；若需整条线路退出，应走 **联动切换**（`coordinated_line_switch`），保证转盘与出水同编号同步备用。
- **转盘饱和率高**: 启动再生程序，切换备用转盘
- **出水毒性超标**: 停止超标管路，切换备用管路（与进水、吸附联动时同线路编号一致）
- **设备故障**: 故障管道切出，备用管道切入

### 4. 定期维护
- 定期轮换运行管道，均衡磨损
- 转盘饱和率≥80%时进行再生
- 每月进行系统性能评估

## 监控界面说明

### 状态显示
1. **运行状态标识**:
   - 🟢 运行中 (running)
   - 🟡 备用/待机 (standby) 
   - 🔴 停止/故障 (stopped)
   - 🟠 维护中 (maintenance)

2. **参数显示**:
   - 运行管道: 显示实时参数值
   - 备用管道: 显示最后记录值（流量为0）

3. **报警规则**:
   - 黄色警告: 参数接近临界值
   - 红色报警: 参数超过阈值
   - 闪烁显示: 需要立即处理

### 控制操作
1. **线路联动切换（首页顶部工具条）**  
   - 位置：三工序监控卡片 **上方** 单行区域，与三张表对应「一条线路跨三列」的布局一致。  
   - 操作：下拉仅列出 **当前运行中的线路**；选择因毒性等原因需 **关闭** 的线路后，点击 **「执行联动切换」**。  
   - **不**再提供「关闭线路 + 启用线路」双下拉；备用线路由系统根据 `line_switch_state` **自动启用**。  
   - 成功后三张表（进水管道 / 吸附转盘 / 出水管道）**同一行编号** 的流量、频率与状态徽章一并刷新。

2. **分工序参数（各列下方控制面板）**  
   - **进水**：仅保留与文档一致的 **毒性、温度、进水流量** 及「应用进水参数」；线路切换不在此重复。  
   - **吸附**：**温度、转盘频率、输入流量、吸附饱和率**（无阀门控制）。  
   - **出水**：**毒性检测、出水流量**（无「深度处理强度」控件）。

3. **应急与追溯**  
   - 联动切换与分工序控制均写入 **控制历史**；联动事件可通过 SSE 订阅 `control_executed`（`action: coordinated_line_switch`）做实时提示。

## 系统优势

1. **高可靠性**: "二备一"设计确保单点故障不影响系统运行
2. **灵活配置**: 可调整运行管道组合，适应不同工况
3. **易于维护**: 备用设备可在不影响生产的情况下进行维护
4. **智能监控**: 实时健康报告和预警机制
5. **可追溯性**: 管路一一对应，便于问题排查

## 配置参数

### 运行模式
- **模式A**: 管道1、2运行，管道3备用
- **模式B**: 管道1、3运行，管道2备用  
- **模式C**: 管道2、3运行，管道1备用
- **自动模式**: 根据设备状态自动选择最优组合

### 切换策略
- **定时切换**: 每24小时自动轮换
- **性能切换**: 根据设备性能指标切换
- **故障/毒性切换**: 检测到故障或毒性超标需切线时，优先使用 **联动切换接口**，保证三工序同线路编号一致
- **手动切换**: 操作员在界面 **线路联动切换** 条中仅选择待关闭运行线，由系统自动启用备用线
