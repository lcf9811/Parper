# 再生、突发加药与 MBR 控制

> **资料对齐**：与 `requirement/三工序系统架构说明.md`、`requirement/规则及约束条件_抽取.md` 及 `scada_plc_data/plc_business_data_model.md`（冷却/MBR、加药、再生热水箱域）一致。若你方在 `requirement/mbr&regen` 下有更细点位表，以现场竣工图与 PLC 表为准，本 Skill 只固定**触发语义与计算口径**。

## 三条逻辑的触发条件（总表）

| 逻辑 | 触发条件（满足即进入对应程序或调节） | 优先级提示 |
|------|--------------------------------------|------------|
| **再生** | 该线 **吸附饱和率 ≥ 80%**（与架构/吸附 Skill 一致）；或调度/审批下达的再生窗口 | 常与 **停该线进水 + 备用线承担负荷** 联动，见 `water-adsorption-saturation` |
| **突发加药** | **进水毒性抑制率已处于「需紧急加药」档**（规则抽取稿：**60%–70%** 为「流量最低 + 启动紧急加药系统」）；或运营定义的 **毒性超标** 位/告警为真 | **≥60%** 时架构要求 **停止进水**，与加药策略并存时以 **安全手册就高**；详见 `water-inlet-toxicity-flow` |
| **MBR** | **再生完成后**（或工艺上）该股水流进入 **冷却/MBR 域**；且 **溶解氧 DO 低于设定下限**（或低于死区下沿） | 与吸附再生 **串行在物料路径** 上：先完成前段再强化 MBR 曝气 |

## 一、再生逻辑

### 1.1 何时触发

- **主判据**：`saturation_pct >= 80`（disk N 与 pipe N 同线）。
- **前置**：由策略或 `water-adsorption-saturation` 已判定须再生；若需切线，**只填 `close_line`**，不手写 `activate_line`（三工序联动）。

### 1.2 执行序列（与你给出的步骤一致）

1. **停止进水**  
   - 对 **待再生线路 N**：将该线 **进水** 置为停止（与架构「一条线路」进水–转盘–出水一致）。  
   - 文档示例：停该线进水后，可启备用线进水，**延时**（如 20 s，以现场联锁为准）待阀门到位。
2. **启动再生**  
   - 在满足联锁（液位、温度许可、泵许可等）后，置位 **再生程序/再生泵/再生回路**（如再生水泵 B04、VA14 等，见 PLC 模型）。
3. **设定参数**（输出到控制侧或建议对象）

| 参数 | 说明 |
|------|------|
| **加药量** | **统一口径**：以 **瞬时流速（m³/h）× 固定时间窗口 120 s** 得到 **120 s 内对应体积基准**，再换算为 **加药泵流量设定或冲程**（与药剂浓度、泵曲线匹配）。公式：`V_120s_m3 = Q_m3h × (120 / 3600) = Q_m3h / 30`。 |
| **转盘频率** | 在 **0–50 Hz** 内按再生阶段设定（勿超硬件上限）；与吸附运行阶段表可区分「再生段」设定。 |
| **再生时间** | 批次时长或步骤总时长（由工艺配方/阶段机定义）。 |
| **再生温度** | 再生热水箱/回路目标温度；须满足设备 **上限 ≤20℃** 的吸附侧约束与再生侧加热器能力（见规则抽取稿，勿顶格）。 |

### 1.3 关联 Skill

- **`water-adsorption-saturation`**：饱和率与换线/审批/即时联动。  
- **`coordinated-line-switch-eval`**：即时联动换线时执行。

## 二、突发加药逻辑（毒性超标）

### 2.1 何时触发

- **规则稿对齐**：进水毒性抑制率 **60%–70%** → **流量最低 + 启动紧急加药系统**。  
- **扩展**：若 SCADA 另有 **「毒性超标」** 布尔或分级（如接近穿透、仪表连锁），在运营配置中 **映射到加药档位** 后同样走本节。

### 2.2 加药量设定

- **基准**：与再生一致，**加药量（体积基准）= 瞬时流速 × 120 s（固定）**  
  - `V_120s_m3 = Q_m3h / 30`。  
- **毒性匹配**：在基准之上乘 **档位系数或查表**（示例：抑制率越高系数越大；具体表由水厂标定，本 Skill 要求输出中带 `toxicity_tier` 与 `dose_multiplier`）。  
- **执行**：紧急加药泵/蠕动泵 **流量设定 + 启停**（加药 1/2/3 或紧急支路，以 PLC 域 `PROC_DOSING` 为准）。

### 2.3 与进水停机的边界

- **抑制率 ≥60%**：架构要求 **停止进水**；加药若为「极低流量下维持药剂接触」的连锁，须在控制策略中与 **停进水** 不冲突（由现场联锁定义）。  
- 智能体侧：**先**满足 `water-inlet-toxicity-flow` 的停机/切线建议，**再**输出加药参数。

## 三、MBR 逻辑（再生后排水进入 MBR）

### 3.1 何时关注

- **工艺位置**：再生后（或主链）出水进入 **冷却/MBR**（见 `PROC_COOLING_MBR`）。  
- **触发调节**：**DO 低于目标下限**（或趋势持续走低穿越死区）。

### 3.2 控制动作

- **监测**：溶解氧（如 `VD3192` 类点位）、必要时跨膜压差、液位（见 `plc_business_data_model.md`）。  
- **动作**：**DO 偏低 → 增大曝气量**（风机转速/阀门开度/曝气支路设定，以现场执行器为准）。  
- **建议**：带 **DO 设定值 + 死区**；调节速率加斜坡，避免与跨膜压差、风量保护冲突。

### 3.3 与其它逻辑的关系

- **再生完成后** MBR 负荷可能变化，宜在再生阶段结束事件后 **提高 MBR 巡检频率** 直至 DO 稳定。

---

## 加药量计算（统一）

```text
Q_m3h     = 当前关联线路瞬时体积流量（m³/h）
T_fixed_s = 120  （固定，秒）

V_basis_m3 = Q_m3h × (T_fixed_s / 3600) = Q_m3h × (120/3600)

实际泵设定 = f(V_basis_m3, 药剂浓度, 泵曲线, toxicity_tier)
```

- **再生**：`f` 主要随再生配方。  
- **突发加药**：`f` 额外随 **毒性档位** 单调不减（就高安全）。
